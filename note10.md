## 跟你的 Compliance Context Infra 的關係

**關聯度：🟡 中高——間接但有意義的客戶**

FullSeam 的 AI agents 做財務作業時，看起來只是「記帳和對帳」，但實際上隱含大量合規判斷：

**1. 收入認列合規（Revenue Recognition）**
- FullSeam 的 AI 合約代理從合約中提取數據，AI 發票代理計算和登帳發票，包括收入認列（revenue recognition）。
- 收入認列必須符合 **ASC 606**（美國）或 **IFRS 15**（國際）。AI agent 怎麼知道「這筆合約的收入應該在何時認列」？需要理解 5 步法的規則。
- 你的 API 查詢範例：`「ASC 606 五步法中，合約修改的收入認列規則是什麼？」`

**2. AP 的稅務合規**
- 處理供應商帳單時，需要判斷：需要代扣稅嗎？1099 表格的門檻是什麼？跨州購買的 sales tax 規則？
- 你的 API 查詢範例：`「向加州供應商支付超過 $600 的服務費，1099-NEC 報告要求是什麼？」`

**3. GL 分類的合規性**
- 費用分類到正確的 GL 科目不只是記帳問題——錯誤分類可能導致稅務合規問題。
- 你的 API 查詢範例：`「根據 IRS Publication 535，這類支出可以作為 business expense 抵扣嗎？」`

**4. 跨境交易**
- FullSeam 服務中大型企業，必然會遇到跨境交易。不同國家的發票要求、增值稅（VAT）、預扣稅規則都不同。
- 你的 API 查詢範例：`「向德國供應商付款，EU 增值稅反向收費機制的適用條件是什麼？」`

**5. TaxProper 背景 = 稅務合規 DNA**
- 創辦團隊的前一家公司就是做房產稅的。他們深知稅務合規的複雜性。FullSeam 擴展到更多財務流程時，合規知識的需求會指數級增長。
